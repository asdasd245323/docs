---
title: "Subscription"
description: "Subscriptions are tied to a user and are created once a user has a plan and SIM. This allows connectivity for their device and SIM."
---

<ResponseField name="object" type="string">
  Type of object is always `subscription`.

  Allowed values: `subscription`
</ResponseField>

<ResponseField name="id" type="string">
  Unique identifier for the subscription.

  Example: `"sub_0SNlurA049MEWV2gSfSxi00xlPIi"`
</ResponseField>

<ResponseField name="metadata" type="Metadata">
  A set of key-value pairs, to store metadata related to the resource.
</ResponseField>

<ResponseField name="activatedAt" type="string | null">
  Time when the subscription was activated.

  Example: `"2021-01-21T19:38:34Z"`
</ResponseField>

<ResponseField name="billing" type="object | null">
  Billing details for the subscription. Present when Billing is enabled on your project.
</ResponseField>

<ResponseField name="canceledAt" type="string | null">
  Time when the subscription was canceled.

  Example: `"2021-01-29T13:22:51Z"`
</ResponseField>

<ResponseField name="cancellationDetails" type="object | null">
  Details about the cancellation of the subscription.

  <Expandable title="properties">
    <ResponseField name="cause" type="string | null">
      Cause of subscription cancellation.

      Optional: This property may not always be included.

      Allowed values: `cancellationRequested`, `deviceBlocked`, `invoiceUnpaid`, `paymentDisputed`, `paymentFailed`, `phoneNumberPortedOut`, `portingExpired`, `simUnusable`, `subscriptionEnded`, `userDeleted`
    </ResponseField>

    <ResponseField name="userComment" type="string | null">
      Comment from the user explaining the cancellation reason. It is only present when the userReason is other.

      Optional: This property may not always be included.
    </ResponseField>

    <ResponseField name="userReason" type="string | null">
      Reason for the user to cancel the subscription. It is only present when the cause is cancellationRequested

      Optional: This property may not always be included.

      Allowed values: `connectivityIssues`, `customerService`, `noDetails`, `other`, `tooExpensive`, `unused`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string">
  Time when the subscription was created.

  Example: `"2021-01-21T19:32:13Z"`
</ResponseField>

<ResponseField name="currentPeriod" type="Period | null">
  Current subscription period. Only present for active subscriptions.

  <Expandable title="properties">
    <ResponseField name="number" type="integer">
      Sequence number of the period.

      Optional: This property may not always be included.

      Example: `1`
    </ResponseField>

    <ResponseField name="start" type="string">
      Beginning of the period.

      Optional: This property may not always be included.

      Example: `"2021-01-21T19:32:13Z"`
    </ResponseField>

    <ResponseField name="end" type="string">
      End of the period.

      Optional: This property may not always be included.

      Example: `"2021-02-20T19:38:34Z"`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="earliestEndAt" type="string | null">
  Earliest time at which the subscription can currently be ended. This depends on the minimum period of the plan. This value is present after the first period has already started, and before the subscription has ended.

  Example: `"2021-02-20T19:38:34Z"`
</ResponseField>

<ResponseField name="endedAt" type="string | null">
  Time when the subscription will end or has ended. This field is populated in different scenarios:
  - When a subscription is canceled, this represents the future date when it will be terminated
  - When a subscription has already ended, this represents the past date when it was terminated
  - For active subscriptions, this field is null

  Example: `"2021-02-20T19:38:34Z"`
</ResponseField>

<ResponseField name="firstUsageAt" type="string | null">
  Time when the subscription was first used.

  Example: `"2021-01-21T19:38:34Z"`
</ResponseField>

<ResponseField name="lastPorting" type="Porting | null">
  Last porting (if any) related with the subscription.
</ResponseField>

<ResponseField name="phoneNumber" type="string | null">
  The phone number attached to the SIM in E.164 string_format. Only available for voice plans.

  Example: `"+19591234567"`
</ResponseField>

<ResponseField name="plan" type="Plan">
  Plans define the network options and parameters offered to the end user, e.g. 'Unlimited data, voice & text'. Plan allowances (data, sms, voice) and limits are configurable, and Gigs will work with you to find the best balance for your users.
</ResponseField>

<ResponseField name="restrictedAt" type="string | null">
  Time when the subscription was restricted.

  Example: `"2021-02-21T19:38:34Z"`
</ResponseField>

<ResponseField name="restrictionDetails" type="object | null">
  Details about the restriction applied to the subscription. Present when the subscription is restricted.

  <Expandable title="properties">
    <ResponseField name="restrictBehavior" type="string">
      Determines how the subscription is restricted. It can have one of the following values:
      - `incomingOnly`: The subscription line can receive calls & SMS, but cannot make calls, send SMS, or use data.
      - `noData`: The subscription line can make and receive calls & SMS, but cannot use data.
      - `throttledData`: The subscription line can still make and receive calls & SMS, and use up to 1GB of data at a reduced speed.

      Allowed values: `incomingOnly`, `noData`, `throttledData`

      Example: `"incomingOnly"`
    </ResponseField>

    <ResponseField name="restoreBehavior" type="string">
      Determines how the subscription is restored. It can have one of the following values:
      - `resetPeriodAnchor`: The subscription cycle is reset to the date of restoration. This means the user has a whole new period to use their plan allowance that was just restored.
      - `keepPeriodAnchor`: The subscription cycle remains unchanged. The next renewal date stays the same as before the restriction.

      Allowed values: `keepPeriodAnchor`, `resetPeriodAnchor`

      Example: `"resetPeriodAnchor"`
    </ResponseField>

    <ResponseField name="cause" type="string">
      Determines the cause that triggered the subscription restriction. It can have one of the following values:
      - `invoiceOverdue`: The subscription was automatically restricted due to an unpaid invoice that has become overdue.
      - `restrictionRequested`: The subscription was manually restricted due to an explicit request.

      Allowed values: `invoiceOverdue`, `restrictionRequested`

      Example: `"invoiceOverdue"`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="sim" type="Sim | null">
  The SIM to which the subscription is bound.
</ResponseField>

<ResponseField name="status" type="string">
  The current status of the subscription. Note that the `initiated` status is currently only available when using the Billing API, where it implies that the subscription's first invoice has not yet been paid.

  Allowed values: `pending`, `initiated`, `active`, `restricted`, `ended`
</ResponseField>

<ResponseField name="user" type="User">
  Users are end consumers of subscriptions. Users can have multiple devices, SIMs, and subscriptions.
</ResponseField>

<ResponseField name="userAddress" type="string | null">
  Current user address identifier (if any) related with the subscription.

  Example: `"adr_0SNlurA049MEWV5ELDmnaqVXgTFT"`
</ResponseField>
